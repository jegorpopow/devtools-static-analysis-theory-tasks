# Добавление новых типов 

## Bool 

Изменим правила генерации уравнений:

Если нужны булевы константы, их придётся отличать от 0 и 1 синтаксически. 
input оставляем целочисленными - если нужно, его можно "распарсить" как 
`input == 0`

|Выражение   | Уравнение |
| --------   | --------- |
|  I         | [\| I \|] = int  |  
|  B         | [\| B \|] = bool | 
|  E1 == E2  | [\| E1 \|] = [\| E2 \|] and [\| E1 == E2 \|] = bool | 
|  E1 == E2  | [\| E1 \|] = [\| E2 \|] and [\| E1 == E2 \|] = bool | 
|  input     | [\| input \|] = int | 
|  X = E     | [\| X \|] = [\| E \|] | 
|if (E) {S}  | [\| E \|] = bool|
|if (E) {S1} else {S2} | [\| E \|] = bool|
| while E {S} | ;[\| E \|] = bool | 
| остальные уравнения не поменялись | |


Точность анализа повысилась: отвергаются те ошибочные программы, в которых 
в условии находится что-то не вычисляющееся в 0 или 1, которые, согласно описанию 
языка являются суррогатом bool-а. 

Полнота анализа упала: мы отвергаем те программы, в которых выражения в условии 
вычилсяются арифметически (корректно - в 0 или 1, но не булевой,а целочисленной арифметикой, вроде $0 \cdot  x + 1$, где $x$ - любая int-овая переменная),
и, например, програмы где одна переменная часть своего времени жизни корректно служит булевой, а часть - корректно служит целочисленной.

## Массивы

В целом - массив можно рассматривать как функцию из целых
чисел в элементы массива. Выход за границы простой анализ тиипов не проверяет, зависимы типов у нас нет, так что это и не возможно относительно разумным образом. 

Добавим новые правила генерации уравнений

|Выражение   | Уравнение |
| --------   | --------- |
|  {}        | [\| {} \|] =  α[]  |  
|  {e1, e2, e3}         | [\| e1 \|] = [\|e2\|] = ... = [\|en\|] and [\| {e1, e2, e3} \|] = [\| e1 \|] []| 
|  E1[E2]  | [\| E1 \|] = [\| E1[E2] \|][] and [\| E2 \|] = int | 
|  E1[E2] = E3 | [\| E1 \|] = [\| E3 \|][] and [\| E2 \|] = int | 

Протипизируем программу (одна строчка обычно порождает много уравнений, но я выписал для каждой результат какой-то простой подстановки их друг в друга / инверсии и убрал дупликаты) . 

```c
main() {
var x,y,z,t;
x = {2,4,8,16,32,64}; // [| x |] = int[]
y = x[x[3]]; // [| y |] [] = [| x |] = int [] => [| y |] = int, 
z = {{},x}; // [| z |] = [| x |][] = int[][], [| {} |] = α[] = [|x|] = int[] => α = int
t = z[1]; // [| t |][] = [| z |] = int[][] => [| t |] = int []
t[2] = y; // [| t |] = [| y |] [] <=> int[] = int[] - выполняется, программа хорошо типизирована
}
```
